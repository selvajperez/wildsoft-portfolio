Los datos no llegan como líneas ordenadas. Llegan como un bosque denso, como un .csv lleno de ramas rotas, inconsistencias y caminos sin señalización.

En WildSoft, el recorrido comienza con un checklist que guía cada paso:

- Se reciben los archivos `.csv` y se cotejan contra los datasets disponibles.
- Se importan a una hoja de cálculo para facilitar la revisión inicial.
- Se validan campos vacíos, tipos de datos y estructuras mal formateadas.

Cada hallazgo es una rama retirada; cada validación, una piedra despejada del sendero. Esta etapa no busca presentar aún, sino preparar el terreno ético y técnico donde después se caminará.
